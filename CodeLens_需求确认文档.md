# CodeLens - 需求确认文档（第三版）

您好！以下是根据我们的沟通整理出的需求确认内容，所有事项均已确认，请您最终过目。

## 项目概述

| 项目 | 确认内容 |
|------|----------|
| 项目名称 | CodeLens |
| 项目定位 | 本地代码上下文检索 MCP Server，供 AI 编程助手调用，替代 Augment 等收费云端方案 |
| 核心优势 | 代码不出本地，安全可控；单文件部署，开箱即用 |
| 开源协议 | MIT License |

## 技术方案

| 项目 | 确认内容 |
|------|----------|
| 开发语言 | Rust |
| 目标平台 | MVP 仅适配 Linux，后续加 Windows + macOS 支持 |
| 分发方式 | 编译为单个可执行文件，MCP 配置中直接指定路径即可使用 |
| MCP 通信方式 | stdio（标准输入输出，Claude Code 等客户端通过这种方式与 MCP Server 交互） |

## 推荐技术栈

| 模块 | 技术 | 说明 |
|------|------|------|
| MCP 协议 | `rmcp` | Rust 的 MCP SDK，处理 JSON-RPC + stdio 通信 |
| Java 解析 | `tree-sitter` + `tree-sitter-java` | 代码解析器，把 Java 代码变成语法树，精确提取类/方法/注解 |
| XML 解析 | `quick-xml` | 高性能 XML 解析库，内存占用极低 |
| BM25 检索 | 自行实现 | BM25 算法本身约 100-200 行代码，自己写更灵活 |
| 文件监听 | `notify` | 监听文件变化，触发增量更新索引 |
| 命令行参数 | `clap` | Rust 最主流的命令行参数解析库 |
| JSON 处理 | `serde` + `serde_json` | Rust 标准的序列化/反序列化库 |
| 日志 | `tracing` | 结构化日志，方便调试 |
| 异步运行时 | `tokio` | Rust 最主流的异步运行时，`rmcp` 依赖它 |

## 架构设计

| 项目 | 确认内容 |
|------|----------|
| 整体架构 | 单一 Server + 多语言模块化设计，一个可执行文件内部按语言分模块 |
| 模块扩展 | 后续可新增语言模块（如 Vue、React 等），无需改动核心检索引擎 |
| 语言筛选 | AI 调用搜索时可指定语言参数（如 `lang="java"`），也可以不指定进行全语言搜索 |

## 支持的文件类型（MVP）

| 项目 | 确认内容 |
|------|----------|
| Java 文件（.java） | 支持解析类、方法、注解、依赖等结构信息，覆盖 Spring Boot + MyBatis 开发场景 |
| XML 文件（.xml） | 支持解析 MyBatis Mapper XML、Android 布局 XML 等配置文件 |
| 后续扩展 | 计划支持 Vue、React 等前端语言 |

## 检索方案

| 项目 | 确认内容 |
|------|----------|
| 检索算法 | BM25 关键词检索（一种根据关键词相关度排序的算法，对代码检索效果好） |
| 语义检索 | MVP 阶段不支持，后续版本考虑加入 |
| 代码结构感知 | 支持，返回类名、方法签名、依赖关系等结构化信息，而非简单的文本行匹配 |

## 索引策略

| 项目 | 确认内容 |
|------|----------|
| MVP 阶段 | 纯内存索引，启动时扫描项目目录建立索引 |
| 后续优化 | 增加磁盘持久化，实现秒启动 |
| 增量更新 | 监听文件变化，自动更新索引（通过 Linux 的 inotify 机制） |

## MCP 工具能力

| 项目 | 确认内容 |
|------|----------|
| 工具 | 搜索代码（`search`）— 根据关键词搜索匹配的代码片段，返回文件路径、行号、上下文代码，支持按语言筛选 |
| 文件摘要 | 不提供，AI 可直接通过 Read 工具查看文件内容 |
| 目录结构 | 不提供，AI 可直接通过 find 等系统命令查看目录 |

> 专注代码内容检索，做到极致。

## 搜索结果策略

| 项目 | 确认内容 |
|------|----------|
| 默认数量 | 返回 Top 10 条最相关结果 |
| 动态调整 | AI 可通过 `limit` 参数自行调整返回数量 |
| 上下文范围 | 由 AI 通过 `context` 参数决定：可选择返回完整代码块（如整个方法体）或匹配行 ± 前后 N 行 |

## 扫描配置

| 项目 | 确认内容 |
|------|----------|
| 扫描范围 | 通过命令行参数指定项目路径，如 `codelens --path /your/project` |
| 忽略规则 | 内置默认规则（自动跳过 `target/`、`build/`、`.git/` 等）+ 读取项目的 `.gitignore` |
