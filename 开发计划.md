## 2. 后端前置开发计划

> 后端全局性、基础性文件，整个项目都会用到的基础设施。

| 序号 | 模块 | 依赖 | 并行组 | 说明 | 状态 |
|------|------|------|--------|------|------|
| 01 | 项目入口与命令行解析 | 无 | P1 | `src/main.rs`：使用 clap 解析命令行参数（`--path` 项目路径），初始化日志（tracing），启动异步运行时（tokio） | ✅ 已完成 |
| 02 | 错误类型定义 | 无 | P1 | 定义全局统一错误类型（使用 thiserror），覆盖 IO 错误、解析错误、索引错误等场景 | ✅ 已完成 |
| 03 | 公共数据模型 | 无 | P1 | `src/models.rs`：定义核心数据结构 — CodeBlock（代码块，包含文件路径、行号范围、代码内容、语言类型、结构信息）、SearchResult（搜索结果，包含匹配代码块、相关度分数、上下文代码） | ✅ 已完成 |
| 04 | Parser trait 定义 | 后端前置03 | P2 | `src/parser/mod.rs`：定义统一的 Parser trait 接口，包含 `parse(file_path) -> Vec<CodeBlock>` 和 `supported_extensions() -> Vec<&str>` 方法，以及语言分发逻辑（根据文件扩展名选择对应解析器） | ✅ 已完成 |
| 05 | 文件扫描与忽略规则 | 后端前置03 | P2 | `src/scanner/`：递归扫描指定目录，内置默认忽略规则（`target/`、`build/`、`.git/`、`node_modules/` 等），读取并解析项目 `.gitignore` 文件，返回待索引的文件列表 | ✅ 已完成 |
| 06 | 内存索引存储骨架 | 后端前置03 | P2 | `src/indexer/store.rs`：定义 IndexStore 结构，提供 `add`、`remove`、`update`、`search` 等基础接口，内部使用 HashMap 按文件路径存储 CodeBlock 集合 | ✅ 已完成 |

## 3. 后端开发计划

> 后端具体业务功能模块。

| 序号 | 模块 | 依赖 | 并行组 | 说明 | 状态 |
|------|------|------|--------|------|------|
| 01 | Java 解析器 | 后端前置04 | P1 | `src/parser/java.rs`：使用 tree-sitter + tree-sitter-java 解析 `.java` 文件，提取类名、方法签名、注解、import 依赖等结构信息，生成 CodeBlock 列表 | ⬜ 未开始 |
| 02 | XML 解析器 | 后端前置04 | P1 | `src/parser/xml.rs`：使用 quick-xml 解析 `.xml` 文件，支持 MyBatis Mapper XML（提取 namespace、select/insert/update/delete 语句及其 id），支持通用 XML 配置文件解析 | ⬜ 未开始 |
| 03 | BM25 检索引擎 | 后端前置06 | P1 | `src/search/bm25.rs`：自行实现 BM25 算法，对索引中的 CodeBlock 进行关键词相关度排序，支持按语言筛选（`lang` 参数），支持 `limit` 参数控制返回数量（默认 Top 10） | ⬜ 未开始 |
| 04 | 索引构建流程 | 后端01, 后端02, 后端03 | P2 | 串联完整索引流程：Scanner 扫描文件 → 根据扩展名分发到对应 Parser → 解析结果写入 IndexStore → BM25 引擎就绪，启动时一次性完成全量索引构建 | ⬜ 未开始 |
| 05 | 文件监听与增量更新 | 后端04 | P3 | `src/indexer/watcher.rs`：使用 notify 库监听项目目录的文件变化事件（创建、修改、删除），触发对应文件的增量索引更新（重新解析变更文件并更新 IndexStore） | ⬜ 未开始 |
| 06 | MCP Server 实现 | 后端04 | P3 | `src/mcp/server.rs`：使用 rmcp 实现 MCP Server，通过 stdio 传输层接收 JSON-RPC 请求，注册 search 工具 | ⬜ 未开始 |
| 07 | search 工具实现 | 后端06 | P4 | `src/mcp/tools.rs`：实现 search 工具的参数解析与响应格式化。参数：`query`（关键词）、`lang`（可选语言筛选）、`limit`（返回数量，默认10）、`context`（上下文模式：完整代码块或匹配行±N行）。返回代码片段（非完整文件），每个片段顶部标注文件绝对路径和行号范围 | ⬜ 未开始 |
| 08 | 端到端集成与测试 | 后端07 | P5 | 全流程集成：命令行启动 → 扫描建索引 → 启动文件监听 → MCP Server 就绪 → 接收 search 请求 → 返回结果。编写集成测试验证完整流程 | ⬜ 未开始 |

## 4. 核对清单

> 项目开发完成后，按需求确认文档逐项验收。

| 序号 | 核对模块 | 对应需求章节 | 说明 | 状态 |
|------|----------|-------------|------|------|
| 01 | 技术方案合规 | 技术方案 | 核对开发语言为 Rust、目标平台为 Linux、分发为单文件、通信方式为 stdio | ⬜ 未开始 |
| 02 | Java 文件解析 | 支持的文件类型（MVP） | 核对 .java 文件能正确解析类、方法、注解、依赖等结构信息，覆盖 Spring Boot + MyBatis 场景 | ⬜ 未开始 |
| 03 | XML 文件解析 | 支持的文件类型（MVP） | 核对 .xml 文件能正确解析 MyBatis Mapper XML、Android 布局 XML 等配置文件 | ⬜ 未开始 |
| 04 | BM25 检索功能 | 检索方案 | 核对关键词检索结果的相关度排序正确性，核对返回结构化信息（类名、方法签名、依赖关系）而非简单文本行 | ⬜ 未开始 |
| 05 | 索引策略 | 索引策略 | 核对纯内存索引、启动时全量扫描、文件变化增量更新（inotify）均正常工作 | ⬜ 未开始 |
| 06 | MCP search 工具 | MCP 工具能力 | 核对 search 工具能根据关键词搜索并返回文件路径、行号、上下文代码，支持按语言筛选 | ⬜ 未开始 |
| 07 | 搜索结果策略 | 搜索结果策略 | 核对默认返回 Top 10、`limit` 参数可调整数量、`context` 参数可切换完整代码块或±N行模式 | ⬜ 未开始 |
| 08 | 扫描配置 | 扫描配置 | 核对 `--path` 参数指定项目路径、内置忽略规则（target/、build/、.git/）、.gitignore 读取均生效 | ⬜ 未开始 |
| 09 | 多语言模块化 | 架构设计 | 核对架构支持按语言分模块，新增语言解析器无需改动核心检索引擎，`lang` 参数筛选正常 | ⬜ 未开始 |
